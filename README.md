# 日本株 自動取得セットアップガイド

毎日18時に自動実行 → Google スプレッドシートに書き込む仕組みです。
初回セットアップは約1〜2時間で完了します。

---

## ファイル構成

```
stock_updater/
├── fetch_stocks.py              # メインスクリプト
├── requirements.txt             # 必要ライブラリ
├── README.md                    # このファイル
└── .github/
    └── workflows/
        └── fetch_stocks.yml     # GitHub Actions 設定
```

---

## セットアップ手順

### ステップ1: Google スプレッドシートを準備

1. Google スプレッドシートを用意（既存のものでOK）
2. **B列の2行目以降**に証券コードを入力しておく（例: `7203.T`）
3. URLから **スプレッドシートID** をコピーしておく
   - 例: `https://docs.google.com/spreadsheets/d/【ここがID】/edit`

---

### ステップ2: Google Cloud でサービスアカウントを作成

1. https://console.cloud.google.com/ にアクセス
2. 新しいプロジェクトを作成（例: `stock-price-updater`）
3. 左メニュー「APIとサービス」→「ライブラリ」
   - 「Google Sheets API」を有効化
   - 「Google Drive API」を有効化
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「サービスアカウント」
5. 名前を入力（例: `stock-bot`）、権限は「編集者」を選択して作成
6. 作成したサービスアカウントをクリック →「キー」タブ →「鍵を追加」→「JSON」
7. ダウンロードされた JSON ファイルの**中身全体**をコピーしておく

---

### ステップ3: スプレッドシートをサービスアカウントと共有

1. ダウンロードした JSON を開き `"client_email"` の値をコピー
   - 例: `stock-bot@stock-price-updater.iam.gserviceaccount.com`
2. Google スプレッドシートを開き「共有」ボタン
3. 上記のメールアドレスを「編集者」として追加して送信

---

### ステップ4: GitHubリポジトリを作成してファイルをアップロード

1. GitHub で新しいリポジトリを作成（Privateでも動作します）
2. 以下のファイルをすべてアップロード
   ```
   fetch_stocks.py
   requirements.txt
   .github/workflows/fetch_stocks.yml
   ```

---

### ステップ5: GitHub Secrets を設定

リポジトリの「Settings」→「Secrets and variables」→「Actions」→「New repository secret」

| Secret名 | 値 |
|----------|----|
| `SPREADSHEET_ID` | ステップ1でコピーしたID |
| `GOOGLE_CREDENTIALS_JSON` | ステップ2でコピーしたJSONの中身全体 |

---

### ステップ6: 動作確認（手動実行）

1. GitHubリポジトリの「Actions」タブを開く
2. 左側の「毎日株価取得」をクリック
3. 「Run workflow」ボタンで手動実行
4. 緑のチェックマークが出ればOK！
5. スプレッドシートを確認してデータが入っていれば完成

---

## 自動実行スケジュール

- **実行時刻**: 毎日 18:00 JST（平日のみ・月〜金）
- 東証の取引終了（15:30）後に十分な余裕を持って実行されます
- 祝日は自動判定していないため、祝日もスクリプトは走りますが、
  yfinance側でデータがない日は取得スキップされます

---

## スプレッドシートの構成

スクリプトはシート名「最新株価」の以下の列を読み書きします。

| 列 | 内容 | 読み書き |
|----|------|---------|
| B | 証券コード（例: 7203.T） | 読み取り |
| F | 終値 | 書き込み（毎日上書き） |
| G | 前日比(%) | 書き込み（毎日上書き） |

> それ以外の列はスクリプトが触らないので、自由に使えます。

---

## 証券コードの形式

証券コードは**4桁の数字 + 取引所サフィックス**の形式です。

| サフィックス | 取引所 |
|-------------|--------|
| `.T` | 東京証券取引所 |
| `.S` | 札幌証券取引所 |
| `.N` | 名古屋証券取引所 |

```
7203.T   # トヨタ自動車（東証）
2764.S   # 銘柄例（札証）
3166.N   # 銘柄例（名証）
```

---

## トラブルシューティング

**Actionsが失敗した場合**
- GitHubの「Actions」タブでエラーログを確認
- Secretsの設定（`SPREADSHEET_ID` / `GOOGLE_CREDENTIALS_JSON`）が正しいか再確認
- サービスアカウントのメールアドレスがスプレッドシートと「編集者」で共有されているか確認
- Google Cloud で「Google Sheets API」と「Google Drive API」が有効になっているか確認

**データが一部取得できない銘柄がある場合**
- 証券コードの末尾に `.T` があるか確認
- その日が休場日の可能性あり（正常動作です）
- 上場廃止済みの銘柄はデータが取得できません
